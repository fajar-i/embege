## 2. AGNOSTIC CORE SYSTEMS (How to use them)
These systems are fully functional, persistent (`DontDestroyOnLoad`), and accessible globally via Singletons or Static classes.

### A. Scene Management (`SceneLoader.cs`)
Handles async loading and loading screens.
*   **Rule:** Never use `SceneManager.LoadScene(string)` directly. Always use the Enum.
*   **Enum:** `SceneList` (Boot, MainMenu, Gameplay, etc.)
*   **Usage:**
    ```csharp
    SceneLoader.Instance.LoadScene(SceneList.MainMenu);
    ```

### B. Audio System (`AudioManager.cs`)
Handles BGM and temporary SFX spawning (to prevent audio cutoffs).
*   **Feature:** Built-in pitch randomizer for juicy UI clicks.
*   **Usage (SFX):**
    ```csharp
    // Plays sound. Set bool to true for slight pitch randomization.
    AudioManager.Instance.PlaySFX("UIClick", true); 
    ```
*   **Usage (BGM):**
    ```csharp
    AudioManager.Instance.PlayBGM("MainTheme");
    ```

### C. Save/Load System (`SaveManager.cs`)
Uses `JsonUtility` to save to `Application.persistentDataPath`.
*   **Data Class:** `SaveData.cs` (Currently holds `bgmVolume` and `sfxVolume`).
*   **Rule:** Modify data in memory, then call save. Loading happens automatically on `Awake`.
*   **Usage:**
    ```csharp
    // Modify memory
    SaveManager.Instance.currentData.sfxVolume = 0.5f;
    // Save to disk
    SaveManager.Instance.SaveGame();
    ```

### D. Event System / Observer Pattern (`GameEvents.cs`)
A global static class using `System.Action` to completely decouple scripts.
*   **Rule:** Publishers must use `?.Invoke()`. Subscribers must Unsubscribe in `OnDisable()`.
*   **Usage (Publishing):**
    ```csharp
    GameEvents.OnMoneyChanged?.Invoke(newMoneyValue);
    ```
*   **Usage (Subscribing):**
    ```csharp
    private void OnEnable() => GameEvents.OnMoneyChanged += UpdateUI;
    private void OnDisable() => GameEvents.OnMoneyChanged -= UpdateUI;
    ```

### E. Object Pooling System (`GenericObjectPooler.cs` & `PoolItem.cs`)
Sistem ini menggunakan `UnityEngine.Pool` (Unity 2021+) untuk manajemen memori yang efisien pada objek yang sering muncul (peluru, partikel, VFX).

* **Rule:** Dilarang keras menggunakan `Destroy()` atau `SetActive(false)` secara manual pada objek hasil pool. 
* **Rule:** Objek harus mengembalikan dirinya sendiri ke gudang melalui komponen `PoolItem`.
* **Usage (Setup):** Pasang `GenericObjectPooler` pada GameObject di Hierarchy, masukkan prefab, dan atur `autoReturnTime` di Inspector.
* **Usage (Spawning):**
    ```csharp
    // Referensi ke spawner di Inspector
    [SerializeField] private GenericObjectPooler vfxPool;

    void TriggerEffect(Vector3 pos) {
        // Mengambil dan mengaktifkan objek di posisi tertentu
        vfxPool.Get(pos, Quaternion.identity); 
    }
    ```
* **Usage (Manual Return):** Jika `useAutoReturn` dimatikan (untuk objek yang mati berdasarkan logika seperti tabrakan):
    ```csharp
    void OnCollisionEnter(Collision collision) {
        // Panggil fungsi return milik komponen PoolItem
        if (TryGetComponent<PoolItem>(out var item)) {
            item.ReturnToPool();
        }
    }
    ```
